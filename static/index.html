Assignment

What The Homework Asks

Build a crawler that starts from an origin URL, crawls links up to depth k, never crawls the same page twice, and controls load with back pressure.

Search must return (relevant_url, origin_url, depth) while indexing is still running, and the UI or CLI must show progress, queue depth, and throttling state.

What We Built

Current Implementation

This solution uses Python standard-library components: urllib, html.parser, threading, queue.Queue, sqlite3, and http.server.

The crawler writes page-by-page into SQLite, so new results become searchable immediately instead of waiting for the whole crawl to finish. The development process itself is documented as a collaboration between architect, backend, frontend, and reviewer agents.
